解析防止方法について

公開日: 2025年11月21日

完全な解析防止はない！

解析を難しくすることは可能だがLuaは専用のスクリプトやツールを使えば簡単に復号化し解析防止コードを捨てれてしまう。
ではどうすれば解析防止を有効に使えるだろうか？

解析防止の正しい使い方

解析防止コードは復号化後のコードに含めると危険ということがわかった。だから解析防止コードはGithubなどから読み込むのが安全であろう。

Githubのファイルの中身の記述をrawで取得するとLuaで実行できる。

Githubからの実行方法

以下のコードで実行するとGithubから取得したコードを実行できる。 


local url = "https://raw.githubusercontent.com/ (自身のURLを入力)"

-- URL取得・読み込み
local response = gg.makeRequest(url)
if not response or not response.content then
    logLua("print('コード取得失敗')")
    gg.alert("コード取得失敗")
    return
end

-- 取得コードを安全に load
local fn, err = load(response.content, "remote_chunk", "bt", _G)
if not fn then
    logLua("print('コード読み込み失敗: "..tostring(err).."')")
    gg.alert("コード読み込み失敗: "..tostring(err))
    return
end

-- 実行
local status, e = pcall(fn)
if not status then
    logLua("print('実行エラー: "..tostring(e).."')")
    gg.alert("実行エラー: "..tostring(e))
else
    logLua("print('コード実行完了')")
end
